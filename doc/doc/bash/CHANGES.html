    This document details the changes between this version, bash-4.4-rc2, and the previous version, bash-4.4-beta2.

    1. Changes to Bash

    1. Fixed an out-of-bounds read in the redirection operator completion code.

    2. Fixed execution context so ‘until continue’ doesn’t disable execution for subsequent commands.

    3. Fixed trap handling code so traps don’t inherit a command’s temporary environment.

    4. Fixed a bug that resulted in incorrect line numbers when a function is defined as part of another function’s execution.

    5. Fixed a bug in the expansion of ${a} in contexts where word splitting is not performed and $IFS is not the default.

    6. Fixed a bug that caused ”“$@” to not expand to an empty argument when there are no positional parameters.

    7. Fixed a bug that caused a shell compiled without job control to use the incorrect exit status for builtin commands preceded by a command executed from the file system that causes the shell to call waitpid().

    8. Improved word completion for quoted strings containing unterminated command substitutions with embedded double quotes.

    1. Changes to Readline

    1. Fixed a bug that caused mode strings to be displayed incorrectly if the prompt was shorter than the mode string.

    1. New Features in Bash

    1. Using ${a} or ${a} with an array without any assigned elements when the nounset option is enabled no longer throws an unbound variable error.

    1. New Features in Readline

    1. New application-callable function: rl_pending_signal(): returns the signal number of any signal readline has caught but not yet handled.

    2. New application-settable variable: rl_persistent_signal_handlers: if set to a non-zero value, readline will enable the readline-6.2 signal handler behavior in callback mode: handlers are installed when rl_callback_handler_install is called and removed removed when a complete line has been read.

    This document details the changes between this version, bash-4.4-beta2, and the previous version, bash-4.4-rc1.

    1. Changes to Bash

    1. Fixed a memory leak when processing ${!var}.

    2. Fixed a bug that caused subshells to free trap strings associated with inherited signals.

    3. Inheriting BASHOPTS from the environment now works to force actions associated with enabling an option, instead of just marking the option as enabled.

    4. Fixed a bug that allowed assignments to BASH_CMDS when the shell was in restricted mode.

    5. Fixed a bug caused by an accidental omission of part of the original patch for EXECIGNORE.

    6. Prompt expansion now quotes the results of the s, h, and H expansions.

    7. Fixed a bug that caused parsing errors in command substitutions with consecutive case statements separated by newlines.

    8. Updated logic used to decide whether bash is running inside an emacs terminal emulator to work with future emacs versions.

    9. Fixed two extended pattern matching bugs caused by premature short- circuiting.

    10. Fixed a memory leak in the code that removes duplicate history entries.

    11. There are a number of bug fixes to coproc, mapfile, declare, unset, and assignment statements that prevent nameref variables from creating and unsetting variables with invalid names.

    12. Fixed a bug that caused variables to be inadvertently marked as both an associative and an indexed array.

    13. Fixed a bug that caused ‘bash -c’ to not run a trap specified in the command string.

    14. There are a number of bug fixes to coproc, mapfile, declare, and assignment statements that prevent nameref variables from overwriting or modifying attributes of readonly variables.

    15. Fixed a bug that caused command substitution to attempt to set the terminal’s process group incorrectly.
